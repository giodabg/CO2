Scontrini - OCR & Receipt Parsing

Introduzione

Il progetto "scontrini" implementa una pipeline locale:

  • acquisizione immagine (foto scontrino)
  • pre-processing (OpenCV)
  • OCR (Tesseract via pytesseract)
  • normalizzazione del testo
  • parsing euristico in strutture dati versionate (ReceiptContractV1)
  • persistenza su SQLite
  • esposizione API via FastAPI (base per UI WebView su Android e Windows)

Architettura (high level)

La pipeline segue il flusso:

Image -> preprocess -> OCR -> postprocess -> parsing -> contract -> storage/API

Moduli principali

  • OCR : pre-processing, OCR engine, post-processing testo
  • Domain : modelli dominio e parsing euristico
  • Storage : schema SQLite, connessione e repository
  • API : API HTTP per ingest e healthcheck
  • CLI : comandi CLI per batch e debug

Qualità e limiti

Il parsing è euristico; gli scontrini hanno formati variabili. Le estensioni tipiche includono:

  • miglioramento regex IT (P.IVA, CF, CAP, aliquote)
  • scoring/confidenza OCR
  • deduplicazione merchant e normalizzazione prodotti
